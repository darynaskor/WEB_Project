# WEB_Project
# Цей застосунок створено для перегляду фільмів, пошуку сеансів, керування улюбленими та організації розкладу показів

## Вимоги

### Обмеження
1. Максимальна кількість фільмів у розкладі — 10
2. Максимальна кількість сеансів на один фільм — 5
3. Максимум 50 активних сеансів усього для одного кінотеатру
4. Максимум 3 активні задачі планування сеансів для адміністратора 

---

### Хід виконання задачі (приклад — планування сеансів)
1. **10%** — перевірка вхідних даних (фільми, час, зали)
2. **30%** — формування початкового розкладу сеансів
3. **60%** — перевірка конфліктів (перетини у часі або залі)  
4. **90%** — збереження результатів у базі даних 
5. **100%** — розклад сеансів сформовано  

---

### Історія задач
Зберігається інформація про:
- назву задачі (наприклад: «Розклад сеансів на тиждень»)
- статус (queued, in progress, done, error, canceled)
- результат (збережений розклад або список сеансів)
- дата та час запуску і завершення

---

### Авторизація
- Логін / Реєстрація (**JWT**)  
- Кожен користувач бачить **лише свої предмети та завдання**  
- Всі запити виконуються через **HTTPS**  

---

### Профіль користувача
- Персональна інформація (ім’я, email)  
- Список обраних фільмів
- Історія заброньованих квитків
- Перегляд майбутніх і минулих сеансів

---

### Робота з предметами та завданнями
- Адміністратор може додавати / редагувати / видаляти фільми (назва, жанр, опис, рік, рейтинг)
- Для кожного фільму можна створювати сеанси (дата, час, зал, ціна)
- Користувач може переглядати всі сеанси та купувати квитки
- Відображається прогрес заповнення залу

---

### Балансування навантаження
- Використання **load balancer** (наприклад, Nginx/HAProxy)  
- Мінімум **2 application servers** для виконання задач
- Єдина база даних для збереження користувацьких даних, фільмів і сеансів
- У разі падіння одного сервера інший продовжує обробку задач
