# WEB_Project
# Застосунок для планування та організації навчання

## Вимоги

### Обмеження
1. Максимальна кількість предметів у розкладі — **10**  
2. Максимальна кількість завдань на один предмет — **5**  
3. Максимум **50 завдань** усього для одного користувача  
4. Максимум **3 активні задачі планування** на одного користувача  

---

### Хід виконання задачі (приклад — планування розкладу)
1. **10%** — перевірка вхідних даних (предмети, час, дедлайни)  
2. **30%** — формування початкового розкладу  
3. **60%** — перевірка конфліктів та оптимізація часу  
4. **90%** — збереження результатів у базі даних  
5. **100%** — розклад сформовано  

---

### Історія задач
Зберігається інформація про:  
- назву задачі (наприклад: «Розклад на тиждень»)  
- статус (`queued`, `in progress`, `done`, `error`, `canceled`)  
- результат (збережений розклад або список завдань)  
- дата та час запуску і завершення  

---

### Авторизація
- Логін / Реєстрація (**JWT**)  
- Кожен користувач бачить **лише свої предмети та завдання**  
- Всі запити виконуються через **HTTPS**  

---

### Профіль користувача
- Персональна інформація (ім’я, email)  
- Список предметів у розкладі  
- Список завдань по кожному предмету  
- Історія виконаних задач  

---

### Робота з предметами та завданнями
- Користувач може **додавати / редагувати / видаляти** предмети (назва, викладач, час)  
- Для кожного предмета можна додавати **завдання** (назва, опис, дедлайн, статус: `to do`, `in progress`, `done`)  
- Користувач бачить загальний список завдань та їхній прогрес  

---

### Балансування навантаження
- Використання **load balancer** (наприклад, Nginx/HAProxy)  
- Мінімум **2 application servers** для виконання задач  
- Єдина база даних для збереження користувацьких даних і результатів  
- У разі падіння одного сервера інший продовжує обробку задач
